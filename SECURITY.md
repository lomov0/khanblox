# Security Policy

## Supported Versions

We take security seriously for the KhanBlox project. The following versions are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.4.x   | :white_check_mark: |
| < 1.4   | :x:                |

## Reporting a Vulnerability

We appreciate the security research community and believe that responsible disclosure of security vulnerabilities helps us ensure the security and privacy of all our users.

### How to Report a Security Vulnerability

If you believe you have found a security vulnerability in KhanBlox, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to:
- **Email**: lomov0@eq.edu.au
- **Subject Line**: [SECURITY] Brief description of the issue

### What to Include in Your Report

To help us better understand the nature and scope of the possible issue, please include as much of the following information as possible:

- Type of issue (e.g., cross-site scripting, injection, unauthorized access)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

This information will help us triage your report more quickly.

### What to Expect

After you submit a report, you can expect:

1. **Acknowledgment**: We will acknowledge receipt of your vulnerability report within 48 hours.

2. **Communication**: We will send you regular updates about our progress. If you have not received a reply to your email within 48 hours, please follow up to ensure we received your original message.

3. **Verification**: We will work to verify the vulnerability and determine its impact.

4. **Resolution**: Once we have verified and assessed the vulnerability:
   - We will work on a fix and plan a release
   - We will keep you informed of our progress
   - We will notify you when the fix is released

5. **Credit**: If you would like, we will publicly acknowledge your responsible disclosure after the vulnerability is resolved (unless you prefer to remain anonymous).

### Security Best Practices for Users

While using KhanBlox, we recommend the following security best practices:

- Keep your browser up to date
- Be cautious when clicking on external links within the platform
- Do not share your personal information with other users
- Report suspicious behavior or content to the maintainers
- Regularly review the project's updates and security advisories

## Preferred Languages

We prefer all communications to be in English.

## Policy

We follow the principle of [Coordinated Vulnerability Disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure).

---

**Note**: This security policy applies to the KhanBlox project. For security issues related to third-party dependencies or platforms that KhanBlox may interact with, please contact those projects directly.
